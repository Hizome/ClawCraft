# ClawCraft 像素资源规范

本文档定义 ClawCraft Web 前端当前阶段的 2D 俯视像素资源标准，目标是统一地图、角色、物件和 UI 资源的制作与导入方式，避免后续返工。

---

## 1. 目标风格

- 方向：2D 俯视像素风
- 参考：宝可梦式城镇探索，而不是横版过关
- 使用场景：
  - Agent 作为 NPC
  - Session 作为房间、建筑或地图区域
  - Chat 作为 NPC 对话
  - Tool / Approval 作为道具店、任务板或交互点

---

## 2. 推荐资源分类

项目资源建议至少分为以下几类：

### 2.1 地图地块 `tileset`

用于拼接地图的基础图块，包括：

- 草地
- 泥地 / 石板路
- 水面 / 水边
- 树木
- 栅栏
- 房屋屋顶 / 外墙 / 门 / 窗
- 地板
- 路牌

### 2.2 角色行走图 `character spritesheet`

用于玩家与 NPC：

- 玩家角色
- Agent 角色
- 商店 NPC
- 任务 NPC

### 2.3 场景物件 `object sprites`

独立摆放的物件，例如：

- 箱子
- 桌椅
- 电脑
- 终端
- 柜台
- 灯
- 传送点

### 2.4 UI 资源 `ui sprites`

用于游戏界面：

- 对话框边框
- 按钮
- 图标
- 头像框
- 选择箭头
- 状态面板

---

## 3. 文件格式规范

### 3.1 图片格式

统一使用：

- `PNG`

原因：

- 支持透明背景
- 无损压缩
- 适合像素图
- Web 和引擎兼容性最好

不建议使用：

- `JPG`：会破坏像素边缘
- `WebP`：可用于最终优化，但不建议作为原始资源格式

### 3.2 地图格式

推荐使用 Tiled 编辑地图，并导出：

- `JSON`

可接受：

- `TMX`

如果 Web 侧用 Phaser，优先 `JSON`，集成更直接。

### 3.3 动画描述

角色动画可以有两种方式：

- 简单阶段：直接在代码中按固定帧宽高切图
- 后续阶段：使用 `JSON + PNG` 描述 spritesheet

---

## 4. 像素尺寸规范

当前推荐统一采用以下标准：

### 4.1 地图 tile 尺寸

- 标准 tile：`16x16`

说明：

- 更接近经典掌机 RPG 的观感
- 地图拼接更灵活
- 更适合俯视场景

如果后续明确希望提升细节，可整体切换到 `32x32`，但整个项目只能保留一套 tile 标准，不要混用。

### 4.2 角色帧尺寸

推荐：

- `16x24`

可接受：

- `32x32`

建议：

- 头部和身体比例夸张一点，符合像素 RPG 识别性
- 保持所有角色的基线一致

### 4.3 缩放规范

渲染时只允许整数倍缩放：

- `2x`
- `3x`
- `4x`

禁止非整数缩放，否则像素边缘会发虚。

---

## 5. 地图制作方式

不要把地图画成一整张背景图。

正确做法是：

- 先画 tileset
- 再用地图编辑器拼地图

### 5.1 推荐地图分层

建议至少包含以下层：

1. `Ground`
   - 草地
   - 路面
   - 水面
   - 地板

2. `Decoration`
   - 花
   - 草丛
   - 小石头
   - 边缘装饰

3. `Object`
   - 树
   - 房屋
   - 栅栏
   - 路牌
   - 桌子

4. `Collision`
   - 不显示
   - 标记不可通行区域

5. `Interaction`
   - 不显示
   - 标记 NPC 触发点、门、任务点、传送点

### 5.2 第一张地图建议内容

建议先做一个最小可用地图：

- 小镇主路
- 一栋 Agent 房子
- 一块水池或草地区域
- 一个路牌
- 一个 Agent NPC
- 一个玩家出生点

这就足够支撑第一版聊天演示。

---

## 6. 角色资源规范

### 6.1 角色朝向

每个角色至少包含 4 个方向：

- 上
- 下
- 左
- 右

### 6.2 行走动画帧数

推荐每个方向：

- `3 帧`

即：

- 站立
- 左脚
- 右脚

如果后续资源更丰富，也可以扩展到 `4 帧`。

### 6.3 推荐排布方式

推荐 spritesheet 排列：

- 每行一个方向
- 每列一个动作帧

例如一个 `16x24` 角色，4 方向 x 3 帧：

- 总尺寸：`48x96`

---

## 7. UI 资源规范

UI 可以分两个阶段做：

### 7.1 当前阶段

可以先用：

- CSS 像素风边框
- 纯色背景
- 少量图标 PNG

### 7.2 后续阶段

导入完整 UI 图：

- 对话框边框图
- 菜单面板图
- 图标 atlas
- 选择箭头
- 任务窗口

如果 UI 边框需要可拉伸，推荐用：

- 9-slice 可切分边框图

---

## 8. 绘制注意事项

### 8.1 像素绘制规则

- 使用硬边像素，不要抗锯齿
- 不要出现模糊边缘
- 只做整数像素移动

### 8.2 调色规则

- 控制颜色数量，不要过多
- 同一类资源保持统一调色板
- 光源方向统一，建议默认左上

### 8.3 地块拼接规则

- tile 边缘必须可无缝拼接
- 路、水、草地的边角要提前设计转角块
- 屋顶、墙体、门框最好拆分成可组合结构

---

## 9. 推荐工具链

推荐使用：

- `Aseprite`：绘制 tileset、角色、物件、UI
- `Tiled`：编辑地图、图层、碰撞、交互点

推荐流程：

1. 在 Aseprite 中绘制 tileset 和角色
2. 导出 `PNG`
3. 在 Tiled 中拼地图并配置图层
4. 导出地图 `JSON`
5. 在 Phaser 中读取 tileset + tilemap

## 11. 当前建议的最小资源清单

为了尽快推进俯视地图版本，第一批资源建议准备：

- `town_tileset.png`
  - 草地
  - 路
  - 水
  - 树
  - 栅栏
  - 房屋基础块

- `player.png`
  - 4 方向
  - 每方向 3 帧

- `agent_01.png`
  - 4 方向
  - 每方向 3 帧

- `props.png`
  - 路牌
  - 花
  - 箱子
  - 桌椅

- `town.json`
  - 主地图

这套资源就足够支持第一版“玩家在城镇中与 Agent NPC 对话”的 ClawCraft Web 前端。

---

## 12. 一句话结论

ClawCraft 当前应统一采用：

- `PNG` 作为原始像素资源格式
- `16x16` 作为基础地图 tile 标准
- `16x24` 作为推荐角色帧尺寸
- `Tiled JSON` 作为地图格式
- tileset 拼图而不是整张背景图

后续所有资源都应围绕这套标准制作和导入。
